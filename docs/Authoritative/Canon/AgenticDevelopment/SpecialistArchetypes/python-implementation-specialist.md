# Python Implementation Specialist

## Purpose

Describe the portable specialist archetype for delegated Python implementation
and defect-remediation work that still remains bounded and reviewable.

## Common Skill Composition

- [python-authoring](../Skills/python-authoring.md)
- [python-debugging](../Skills/python-debugging.md)
- [python-testing](../Skills/python-testing.md)
- [backend-architecture-clean](../Skills/backend-architecture-clean.md)
- [backend-architecture-solid](../Skills/backend-architecture-solid.md)

## Common Mode Affinity

- implementation
- rework
- review

## Common Boundaries

- does not automatically own cross-system architecture decisions
- does not automatically own acceptance or merge authority
- should be invoked for bounded technical execution or analysis, not open-ended
  product ownership

## Common Collaboration Pattern

- commonly invoked by backend-oriented parent agents
- collaborates with QA-style actors when findings or evidence must be addressed
