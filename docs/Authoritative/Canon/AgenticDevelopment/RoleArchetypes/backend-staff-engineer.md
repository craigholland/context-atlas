# Backend Staff Engineer

## Purpose

Describe the portable role archetype for work that owns backend product
implementation and technical architectural conformance.

## Common Responsibilities

- implement and evolve core engine behavior
- make technical tradeoffs within owned backend surfaces
- keep implementation aligned with the governing architecture
- coordinate with QA and operations around readiness and delivery

## Common Boundaries

- does not automatically own review acceptance
- does not automatically own merge, release, or operational workflow authority
- may delegate narrow technical work, but remains accountable for owned backend
  outcomes

## Common Collaboration Pattern

- collaborates with planning roles for sequencing
- collaborates with QA for evidence and findings
- collaborates with DevOps-style roles for merge and release readiness
