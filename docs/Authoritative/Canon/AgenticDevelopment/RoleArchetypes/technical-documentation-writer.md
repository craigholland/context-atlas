# Technical Documentation Writer

## Purpose

Describe the portable role archetype for work that owns user-facing technical
explanation, onboarding, and operational clarity.

## Common Responsibilities

- write and maintain guides, onboarding material, and product-facing
  explanations
- improve how users understand and operate the current product surface
- translate technical capability into clear user-facing documentation

## Common Boundaries

- does not automatically own backend implementation authority
- does not automatically act as QA or operational approver
- may collaborate closely with implementation roles, but remains focused on
  explanation and user-facing clarity

## Common Collaboration Pattern

- collaborates with implementation roles to keep docs aligned with reality
- collaborates with evaluation or QA roles where documentation is part of
  product acceptance
